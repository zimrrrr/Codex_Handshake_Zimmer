# Workspace Demo Readiness

## Workflow

1. Validate the primary demo path from `Agenda` to cross-surface management.
2. Remove dead ends, unclear CTAs, and noisy UI.
3. Ensure seeded and placeholder-backed context tell a coherent story.
4. Tighten visual and copy consistency on key screens.
5. Prefer improvements that make usefulness obvious fast.

## Demo Checklist

- [ ] Agenda immediately communicates "what matters now"
- [ ] School, Work, and Projects each have a distinct purpose
- [ ] Agent suggestions feel useful and user-controlled
- [ ] Trust and autonomy boundaries are visible in the UI
