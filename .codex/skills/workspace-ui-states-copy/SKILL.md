# Workspace UI States And Copy

## Workflow

1. Define loading, empty, error, and success states for each async or data-dependent surface.
2. Write concise copy that explains purpose and next action.
3. Make placeholder-backed data feel intentional and visible.
4. Make trust boundaries explicit in copy and controls.

## Copy Rules

- Never imply autonomous external actions.
- Never imply outbound email sending.
- Use explicit confirmation language for app-local mutations.
- Prefer clear operational language over vague AI marketing copy.

## Validation

- [ ] Empty states explain what the view is for
- [ ] Failure states are clear and actionable
- [ ] Placeholder sources are visibly intentional
- [ ] Trust boundaries are understandable without external docs
